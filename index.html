1 Introducción

Este software de gestión digital de préstamos de libros es un sistema desarrollado para facilitar el proceso de préstamo y seguimiento de libros de la biblioteca pública departamental. El sistema permitirá a los usuarios registrarse, iniciar sesión, buscar libros por título o autor, solicitar préstamos de libros disponibles, renovar préstamos activos y recibir recordatorios sobre los vencimientos. También podrán consultar su historial de préstamos y visualizar bloqueos en caso de deudas o retrasos. Por otra parte, el administrador tendrá acceso a reportes de uso para una mejor toma de decisiones. Todo esto con el fin de lograr la satisfacción del cliente a través de un programa que agiliza los distintos procesos.

1.1 Propósito

Su propósito es definir los requisitos funcionales y no funcionales del sistema, para mostrar de manera detallada a los clientes, el equipo de desarrollo y administradores del sistema su alineación con las funcionalidades y las limitaciones del proyecto, así como todo el proceso de diseño, desarrollo, pruebas y validación, con el fin de que todos estén bien comunicados sobre lo que el sistema debe poder hacer.

1.2 Alcance

El software permitirá la gestión digital del préstamo de libros dentro de la biblioteca. Donde, el sistema estará disponible para usuarios registrados, quienes podrán buscar libros por título o autor, solicitar préstamos, renovar libros, consultar su historial y recibir notificaciones de vencimiento. Además, mostrará bloqueos en caso de retrasos o deudas pendientes. Para el administrador, el sistema ofrecerá herramientas para consultar reportes de uso y supervisar la actividad de los usuarios.

1.5 Visión general del documento

El documento está organizado en varias secciones que explican de los aspectos más importantes del sistema. Comienza con una introducción donde se describe el objetivo del software para gestionar la préstamos de libros en la biblioteca pública departamental. Luego, viene el propósito del documento, que es definir claramente qué debe hacer el sistema y cuáles son sus características principales. También se explica el alcance, definiendo las funciones principales que podrán usar tanto los usuarios registrados como los administradores. Después, se presenta una descripción general del sistema, incluyendo cómo interactúan los usuarios con la plataforma y cuáles son los requisitos específicos que debe cumplir el software para funcionar bien.

Y por último estos requisitos estarán documentados siguiendo un procedimiento que especifica: un código identificador (por ejemplo, RF-001 o RNF-001), una descripción clara y específica, el tipo de requisito (funcional o no funcional), su prioridad de importancia (alta, media o baja), prioridad de urgencia (alta, media o baja), el estado del desarrollo (en construcción, pendiente de verificación, pendiente de validación o validado) y la fuente del requerimiento (usuario, administrador o sistema). Cada sección tiene como finalidad garantizar una comprensión completa del funcionamiento, diseño, desarrollo, pruebas y validación del sistema, asegurando así que todos los involucrados tengan una visión clara y compartida de lo que se debe implementar.

2 Descripción general

Book central es un software que sirve para que los usuarios de la biblioteca pública departamental puedan registrarse, iniciar sesión. Al ingresar a la página web los usuarios pueden buscar libros que están en la biblioteca viendo su resumen, autor y editorial para saber de qué trata cada uno. También pueden solicitar el préstamo de los libros que quieran usando datos como título, autor, estante, contacto y documento de identidad, y aparte hay una parte donde se puede renovar el préstamo que ya tienen.

Luego, el sistema muestra un historial con las fechas de préstamo, devolución y el estado en que está cada libro, y también deja ver si el usuario tiene alguna deuda o restricción activa. En ese mismo lugar el usuario puede ver los pagos pendientes y saber hasta cuándo tiene el veto. Por último, este también permite el acceso a reportes y supervisión de actividades hecho para los administradores.

2.1 Perspectiva del producto

La página web de Book central es un producto que funciona de manera independiente porque solo se encarga de gestionar los préstamos de libros en la biblioteca pública departamental y no depende de otro sistema más grande para poder trabajar. Sirve solo para que los usuarios puedan registrarse, iniciar sesión, buscar los libros, pedirlos prestados y renovar los que ya tienen, además de consultar su historial y ver si tienen bloqueos o deudas. Aunque podría conectarse a otras partes como bases de datos de la biblioteca o sistemas de reportes, pero en sí el software está pensado para que funcione solo como la parte que hace todo el control del préstamo de libros que tiene la biblioteca.

2.4 Restricciones

El software deberá cumplir con la norma ISO 27001 (Sistema de Gestión de Seguridad de la Información), para la seguridad de la información. Este funcionará en dispositivos con navegadores web y conexión a internet estable. Se necesitará la dirección de correo electrónico para el envío de recordatorios. También el administrador será el único con permisos a datos confidenciales, el sistema un no tiene definido en que entorno se desarrollará.

2.5 Suposiciones y dependencias

El sistema dependerá de que el usuario tenga acceso a navegadores web con conexión a internet estable. Por otra parte como suposiciones, el sistema utilizará el correo electronico para el envio de notificaciones y que los actores del sistema no cambiarán a futuro. Si este recibe cambios deberá revisarse para poder ajustar nuevamente los requisitos del software.
